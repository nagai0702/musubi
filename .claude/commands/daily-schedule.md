# 朝の工程表作成スキル

毎朝の工程表を作成する。以下の手順で実行すること。

## 実行手順

1. **Google Calendarから今日の予定を取得**
   - 連携済みカレンダーすべてから本日の予定を取得
   - 開始時刻・終了時刻・タイトル・場所を取得

2. **DAILY.mdを読み込む**
   - 昨日の未完了タスクを確認
   - 持ち越しタスクをリストアップ

3. **03_projects/ フォルダを確認**
   - 各プロジェクトフォルダ内の進行状況を把握
   - 期限の近いタスクを特定

4. **タスクを分類**
   - 「緊急度×重要度」のマトリクスで分類
   - 認知負荷（高・中・低）を付与
   - 会議やミーティングの間の空き時間を計算

5. **15分刻みの工程表を生成**
   - カレンダーの予定をベースに配置
   - 空き時間にタスクを優先度順に割り当て
   - 集中作業は午前に、軽作業は午後に配置
   - 休憩時間も含める

6. **DAILY.mdに本日の工程表を書き込む**
   - 日付セクションを新規作成（既存の場合は更新）
   - 工程表テーブルを記載
   - 今日の重点タスクを3つ以内でハイライト

## 出力フォーマット

```markdown
## YYYY-MM-DD (曜日)

### 今日の重点タスク
1. [最重要タスク]
2. [重要タスク]
3. [重要タスク]

### 工程表
| 時間 | タスク | カテゴリ | 優先度 | 認知負荷 |
|:-----|:------|:--------|:------|:---------|
| 09:00-09:15 | メール確認 | 事務 | - | 低 |
| 09:15-10:00 | [タスク名] | [カテゴリ] | [高/中/低] | [高/中/低] |
| ... | ... | ... | ... | ... |

### 昨日からの持ち越し
- [ ] [タスク名]

### 本日の予定（カレンダー）
- HH:MM - HH:MM [予定名] @[場所]
```

## 注意事項
- ユーザーに「おはようございます」と挨拶してから工程表を提示する
- カレンダーの予定は変更不可として扱い、空き時間にタスクを配置する
- 予定が過密な日は、無理にタスクを詰め込まない
